## 🎯 图2：能力边界划分（核心图）

```

│                    合约SDK能力边界（L3）                                   │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ╔══════════════════════════════════════════════════════════════════╗  │

│  ║  Token模块：                                                     ║  │
│  ║    • Transfer()   → 1 input + 2 outputs                          ║  │
│  ║    • Mint()       → 0 inputs + AssetOutput                        ║  │
│  ║    • Burn()       → N inputs + 0 outputs                           ║  │
│  ║    • Approve()    → MultiKeyLock / DelegationLock                 ║  │
│  ║    • Freeze()     → TimeLock / HeightLock                         ║  │
│  ║    • Airdrop()    → N inputs + M outputs (批量)                   ║  │
│  ║                                                                  ║  │
│  ║  Staking模块：                                                   ║  │
│  ║    • Stake()      → N inputs + M outputs + ContractLock          ║  │
│  ║    • Unstake()    → ContractLock UTXO + 解锁                     ║  │
│  ║    • Delegate()   → DelegationLock                                ║  │
│  ║                                                                  ║  │
│  ║  Governance模块（基础）：                                         ║  │
│  ║    • Propose()    → StateOutput (提案内容)                        ║  │
│  ║    • Vote()       → MultiKeyLock / ThresholdLock                   ║  │
│  ║                                                                  ║  │
│  ║  Market模块（基础）：                                             ║  │
│  ║    • Escrow()     → ContractLock (托管)                           ║  │
│  ║    • Vesting()    → TimeLock + HeightLock (分阶段释放)            ║  │
│  ║                                                                  ║  │
│  ╚══════════════════════════════════════════════════════════════════╝  │
│                                                                          │
│  ╔══════════════════════════════════════════════════════════════════╗  │
│  ║  ❌ SDK不提供：组合场景（业务编排）                                 ║  │
│  ╠══════════════════════════════════════════════════════════════════╣  │
│  ║                                                                  ║  │
│  ║  DeFi组合场景：                                                   ║  │
│  ║    ❌ AMM交换逻辑（价格计算、滑点保护）                             ║  │
│  ║    ❌ 流动性池管理（份额计算、比例管理）                             ║  │
│  ║    ❌ 借贷逻辑（利率计算、清算机制）                                 ║  │
│  ║                                                                  ║  │
│  ║  NFT组合场景：                                                    ║  │
│  ║    ❌ NFT拍卖逻辑（竞价、时间管理）                                 ║  │
│  ║    ❌ NFT市场逻辑（版税、手续费计算）                               ║  │
│  ║                                                                  ║  │
│  ║  游戏组合场景：                                                   ║  │
│  ║    ❌ 游戏战斗逻辑（回合制、随机数）                                 ║  │
│  ║    ❌ 游戏经济系统（货币、道具交易）                                 ║  │
│  ║                                                                  ║  │
│  ║  治理组合场景：                                                   ║  │
│  ║    ❌ 投票统计逻辑（计数、阈值判断）                                 ║  │
│  ║    ❌ DAO治理流程（提案→投票→执行）                                 ║  │
│  ║                                                                  ║  │
│  ╚══════════════════════════════════════════════════════════════════╝  │

│                                                                          │

### 场景5：RWA - 现实世界资产代币化平台

```
┌──────────────────────────────────────────────────────────────────────────┐
│  应用场景：RWA代币化平台                                                   │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  【完整应用】                                                             │
│  ┌────────────────────────────────────────────────────────────────┐    │
│  │  TokenizeAsset(assetInfo)                                      │    │
│  │  {                                                              │    │
│  │    // 1. 应用层：资产验证逻辑                                    │    │
│  │    if !verifyAsset(assetInfo) {                                 │    │
│  │      return ERROR_VERIFICATION_FAILED                          │    │
│  │    }                                                            │    │
│  │                                                                 │    │
│  │    // 2. SDK：基础能力                                          │    │
│  │    token.Mint(owner, assetTokenID, totalSupply) ← SDK提供      │    │
│  │                                                                 │    │
│  │    // 3. 应用层：元数据管理                                      │    │
│  │    storeAssetMetadata(assetTokenID, assetInfo)                 │    │
│  │  }                                                              │    │
│  │                                                                 │    │
│  │  DistributeYield(assetTokenID, yieldAmount)                    │    │
│  │  {                                                              │    │
│  │    // 1. 应用层：收益计算逻辑                                    │    │
│  │    shares := getTotalShares(assetTokenID)                       │    │
│  │    perShare := yieldAmount / shares                             │    │
│  │                                                                 │    │
│  │    // 2. SDK：基础能力                                          │    │
│  │    for holder in holders {                                     │    │
│  │      amount := perShare * holder.shares                        │    │
│  │      token.Transfer(pool, holder, paymentToken, amount) ← SDK  │    │
│  │    }                                                            │    │
│  │  }                                                              │    │
│  └────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  【能力分解】                                                             │
│  ┌────────────────────────────────────────────────────────────────┐    │
│  │  ✅ SDK提供：                                                    │    │
│  │     • token.Mint() - 资产代币化                                 │    │
│  │     • token.Transfer() - 所有权转移                             │    │
│  │     • market.Escrow() - 资产托管                                │    │
│  │     • market.Vesting() - 分阶段释放                              │    │
│  │                                                                 │    │
│  │  ❌ 应用层实现：                                                  │    │
│  │     • verifyAsset() - 资产验证逻辑                               │    │

## 💡 核心结论

### 1. 合约SDK应该提供：基础能力（原子操作）

**标准**：
- ✅ 可以映射为单一的输入输出组合模式
- ✅ 是其他复杂场景的基础构建块
- ✅ 不包含特定业务逻辑

**示例**：
- `token.Transfer()` → `1 input + 2 outputs`
- `token.Mint()` → `0 inputs + AssetOutput`
- `staking.Stake()` → `N inputs + M outputs + ContractLock`

### 2. 合约SDK不应该提供：组合场景和应用场景

**标准**：
- ❌ 包含复杂的业务逻辑（价格计算、利率计算、游戏规则等）
- ❌ 需要多个基础能力的组合
- ❌ 不同应用有不同的实现方式

**示例**：
- AMM交换（需要价格计算、滑点保护）
- NFT拍卖（需要竞价逻辑、时间管理）
- 游戏战斗（需要战斗逻辑、随机数）

### 3. 应用场景的实现方式

**方式1**：应用层直接使用SDK基础能力
```go
// 应用层实现
func Swap() {
    price := calculateAMMPrice(...)  // 应用层逻辑
    token.Transfer(...)              // SDK基础能力
    checkSlippage(...)               // 应用层逻辑
}
```

**方式2**：业务编排层提供组合能力（链下SDK）
```go
// 业务编排层实现
func (s *MarketService) SwapAMM() {
    price := s.calculateAMMPrice(...)  // 业务编排层逻辑
    // 调用合约SDK基础能力
    // UTXO选择、费用计算、签名（链下SDK完成）
}
```
